# AI Decision Support System for Supply Chains

> Система поддержки принятия решений для управления складскими запасами с интеграцией искусственного интеллекта

---

## О проекте

**AI Decision Support System for Supply Chains** — система, которая в реальном времени отслеживает состояние складских запасов, автоматически выявляет риски, классифицирует их по приоритету и предлагает конкретные действия. Вместо ручного мониторинга десятков позиций система сама сигнализирует о проблемах — специалист фокусируется только на нестандартных ситуациях.

Проект создан как практическое исследование применения ИИ в логистике и управлении цепями поставок.

---

## Как работает система

Склад передаёт данные: остаток товара, дневной расход, статус поставки, информация о поставщике. Система последовательно обрабатывает их через несколько модулей и выдаёт приоритизированный список алертов с готовыми рекомендациями.

```
Node Simulator → Metrics Engine → Rule Engine → Decision Engine → AI Layer → Dashboard
```

### Модули

| Модуль | Что делает |
|--------|-----------|
| **Node Simulator** | Имитирует складские узлы, генерирует и отправляет данные |
| **Metrics Engine** | Считает `days_of_stock` и `risk_score` по каждому узлу |
| **Rule Engine** | Применяет бизнес-правила, создаёт алерты с приоритетом P1–P4 |
| **Decision Engine** | Подбирает готовое решение из playbook типовых сценариев |
| **AI Layer** | Claude API объясняет алерты на русском, отвечает на вопросы оператора |
| **Dashboard** | Streamlit-интерфейс: счётчики, карточки рекомендаций, AI-чат |

### Логика алертов

| Уровень | Условие | Приоритет |
|---------|---------|-----------|
| 🔴 CRITICAL | Запас менее 3 дней | P1 |
| 🟠 HIGH | Задержка поставки более 3 дней | P2 |
| 🟡 MEDIUM | Поставщик задерживал 2+ раза | P3 |
| 🟢 LOW | Штатная ситуация | P4 |

## Запуск локально

### Требования

- Python 3.10+
- API-ключ Anthropic (Claude API)

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/whoiscarolin/supply-chain-ai.git
cd supply-chain-ai

# Создать виртуальное окружение
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt
```

### Настройка

Создать файл `.env` в корне проекта:

```env
ANTHROPIC_API_KEY=_api_key_here
```

### Запуск

```bash
# 1. Запустить API-сервер
uvicorn main:app --reload

# 2. В отдельном терминале — запустить дашборд
streamlit run dashboard.py

# 3. В отдельном терминале — запустить симулятор узлов
python node_simulator.py
```

Дашборд откроется по адресу `http://localhost:8501`
